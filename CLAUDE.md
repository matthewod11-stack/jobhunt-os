# jobhunt-os

## What this is

Your job-search workspace. Clone it, run `/setup`, and work your search from here. Every session loads this file, so Claude always knows your profile, your materials, and where everything lives. The workspace ships with a fictional example persona (Jordan Reyes) so you can see what good output looks like — `/setup` replaces all of it with yours.

## Structure

- `.claude/commands/` — the 7 commands
- `profile/` — voice.md + fit-profile.json (yours, generated by /setup)
- `corpus/` — answer-bundles.md, cheat-sheet.md, question-trends.md (seeded by /setup, grown by /debrief and /recap)
- `templates/` — resume variant sources + build-resume.sh + resume.css
- `applied/` — tailored resumes and cover letters per company
- `interview-prep/` — prep docs, debriefs, recaps, outreach notes
- `docs/` — SOURCING-PLAYBOOK.md, WORKFLOW.md, vc-registry.md
- `tracker.csv` — the single log of every company/application/touch

## Commands

| Command | Purpose | Key inputs |
|---------|---------|------------|
| `/setup` | One-time interview; generates your profile, corpus, and resume variants (replaces the Jordan Reyes examples) | Your resume(s), writing samples, career stories |
| `/scout` | Source new opportunities from funding news + VC portfolios, scored against your fit profile | profile/fit-profile.json, docs/vc-registry.md |
| `/apply {jd-url}` | Tailor a resume variant to a job posting, build the PDF, optionally draft a cover letter, log to tracker | JD URL, templates/, profile/voice.md |
| `/outreach {company} [person]` | LinkedIn-scoped notes: 300-char connection note, short DM, warm follow-up | Tracker row, corpus, voice |
| `/prep {company} [interviewer] [jd-url]` | Interview prep doc: research + positioning + likely questions + landmines | Corpus, prior prep/debriefs, applied resume |
| `/debrief {company} {round}` | Post-interview guided debrief; promotes reusable lessons to the corpus | Transcript or your recollections, prior prep doc |
| `/recap {company} [outcome]` | Whole-arc retrospective when a process ends | All prep/debrief docs for the company |

## The loop

`/setup` seeds `profile/` and `corpus/` once. `/scout` finds companies worth your time. `/apply` and `/outreach` act on them. `/prep` readies you for each conversation. `/debrief` and `/recap` write what you learned back into the corpus, so the next prep starts smarter than the last. The corpus is why this gets better the longer you use it.

## Modes of work

1. **Sourcing** — finding and scoring companies (`/scout`, `/outreach`)
2. **Applying** — tailoring materials for a specific posting (`/apply`)
3. **Prepping** — readying for a specific conversation (`/prep`)
4. **Reflecting** — processing what happened (`/debrief`, `/recap`)

Ask which mode at session start if unclear.

## Conventions

- File naming: tailored resumes go in `applied/` as `{Your Name} {Company} Resume.md` and cover letters as `{Your Name} {Company} Cover Letter.md`; prep docs in `interview-prep/` as `{company}-{context}.md`; debriefs as `{company}-post-call-{N}.md`; recaps as `{company}-recap.md`; outreach notes as `{company}-outreach.md`
- Never overwrite prep docs — add sections or create new files for new rounds
- `tracker.csv` is the single log; every command that acts on a company updates it
- Example content (the Jordan Reyes persona) carries the header `<!-- Example output — /setup will replace this with yours. -->` — it's there to show what good output looks like (JSON files like fit-profile.json can't carry the comment; /setup replaces them all the same)
